# Pack doctrine, profondeur d'épisode GDIY

Version 2, juillet 2026. Auteur : Matthieu Stefani. Grille PERMANENTE : chargée
dans les prompts de génération (src/lib/fiche/generation.ts) et à charger avant
tout challenge de fiche.

## L'essentiel

- Extraire le SYSTÈME, pas l'histoire : le mode opératoire reproductible.
- Trois familles de mécaniques : action (le geste, la routine), réflexion
  (décider sous incertitude), innovation (la singularité vs les pairs).
- Trois couches : A mécanique personnelle (60 % antenne, priorité absolue),
  B état de l'art du domaine (20 %, SUBORDONNÉE à A, arme l'intervieweur),
  C leçon transférable (20 %, explicitement nommée, jamais implicite).
- Archétypes : fondateur (défaut), dirigeant de groupe coté (ascension et
  pouvoir à l'échelle), artiste (processus créatif), sportif (entraînement,
  pression), avocat (dossier, audience, incertitude du verdict),
  médecin/scientifique (décision sous incertitude, rapport à l'erreur),
  politique (neutralité, méthode, pas la polémique). Hybride : croiser les
  grilles, chercher la tension entre elles.
- Distinction sectorielle : identifier les termes adjacents que le grand
  public confond (ex. biopharma ≠ MedTech), les poser dans la fiche.
- Arbitrage de densité : si plus de 3 questions sur 10 portent sur le domaine
  et non sur la personne, rééquilibrer.
- Ton : profondeur sur le fond, légèreté sur la forme. Le fond dense vit dans
  les 10 questions, la légèreté dans les questions clips.
- Test de qualité d'une question : si la réponse pourrait figurer dans un
  article déjà publié, la question est faible. Exiger un critère, un seuil,
  un arbitrage, un cas précis.

Le document source complet (avec l'exemple travaillé Tordjman et l'annexe
biopharma 2026) est conservé dans les uploads de session et résumé ici ;
les règles ci-dessus sont celles injectées dans le code.
